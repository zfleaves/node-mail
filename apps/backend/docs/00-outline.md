# 系统流程大纲

## 一、整体架构

### 1.1 技术栈
- **后端框架**: Express.js
- **数据库**: MySQL (主要数据存储) + Redis (会话管理 + 库存缓存)
- **认证**: JWT (7天有效期) + Redis 会话 (15分钟滑动过期)
- **文件存储**: 本地文件系统 (Multer)
- **并发处理**: Redis 原子操作 + 子进程异步处理

### 1.2 数据流
```
客户端 → 认证中间件 → 权限验证 → 控制器 → 数据访问层 → 数据库/Redis
                  ↓
            子进程异步处理
```

---

## 二、认证与会话流程

### 2.1 登录流程
1. 用户提交用户名/密码
2. 验证用户凭证
3. 生成 JWT Token (7天有效期)
4. 创建 Redis 会话 (15分钟 TTL)
5. 返回 Token 给客户端

### 2.2 请求认证流程
1. 客户端携带 JWT Token 请求
2. 验证 JWT 签名和有效期
3. 检查 Redis 会话是否存在
4. 刷新 Redis 会话 TTL (滑动过期)
5. 解析用户信息到 `req.user`

### 2.3 登出流程
1. 删除 Redis 会话
2. 客户端清除本地 Token

---

## 三、用户管理流程

### 3.1 创建用户
1. 接收用户数据
2. 验证输入（用户名、密码、邮箱）
3. 检查用户名/邮箱重复
4. 密码加密存储
5. 返回创建结果

### 3.2 用户权限体系
- **admin**: 完全权限
- **manager**: 用户管理（部分）、商品管理、订单管理
- **staff**: 商品只读、订单部分更新

---

## 四、产品管理流程

### 4.1 创建产品（带文件上传）
1. 接收产品数据 + 文件
2. 验证产品信息
3. 上传文件到本地存储
4. 处理价格（计算折扣率）
5. 记录价格历史
6. 缓存库存到 Redis
7. 存储产品信息

### 4.2 更新产品
1. 验证权限
2. 更新产品信息
3. 处理文件变更
4. 记录价格历史（如有变化）
5. 同步库存到 Redis

### 4.3 删除产品
1. 获取产品信息
2. 删除关联文件
3. 删除数据库记录

### 4.4 价格管理
- **价格快照**: 订单创建时保存当前价格
- **价格历史**: 记录所有价格变更
- **价格稳定性**: 分析价格波动和趋势

---

## 五、订单管理流程

### 5.1 创建订单（高并发场景）
1. 接收订单数据
2. **使用 Redis 原子操作扣减库存**
   - 检查 Redis 库存
   - 原子操作扣减库存
   - 记录扣减的商品
3. **数据库事务创建订单**
   - 插入订单记录
   - 插入订单明细
   - 价格快照
4. **异步更新数据库库存**（不阻塞订单创建）
5. **发送消息给子进程**进行后续处理
6. 返回订单信息

### 5.2 订单异常处理
- 库存不足：回滚 Redis 库存
- 数据库错误：回滚 Redis 库存

### 5.3 取消订单
1. 查询订单信息
2. **使用 Redis 原子操作恢复库存**
3. 更新订单状态
4. **异步更新数据库库存**

### 5.4 子进程处理
- **订单创建后处理**: 发送通知、记录日志等
- **过期订单清理**: 定时扫描并取消过期订单

---

## 六、文件管理流程

### 6.1 文件上传
1. 验证文件类型（图片/视频）
2. 验证文件大小（图片 10MB，视频 500MB）
3. 生成唯一文件名
4. 保存到本地存储
5. 返回文件访问 URL

### 6.2 文件访问
- 静态文件服务：`/uploads/:filename`
- 文件信息查询：`/api/uploads/info/:filename`
- 存储统计：`/api/uploads/stats`

---

## 七、库存同步流程

### 7.1 缓存预热
- 系统启动时加载所有产品库存到 Redis
- 设置 1 小时过期时间

### 7.2 库存同步
- **Redis → 数据库**: 定期同步或手动触发
- **数据库 → Redis**: 重新加载或手动触发

### 7.3 高并发处理
- 使用 Redis 原子操作避免竞争条件
- 异步更新数据库，避免阻塞订单创建

---

## 八、系统启动与关闭

### 8.1 启动流程
1. 连接数据库
2. 初始化数据库表结构
3. 连接 Redis
4. 预热库存缓存
5. 启动子进程
6. 启动 HTTP 服务器

### 8.2 优雅关闭
1. 接收关闭信号（SIGTERM/SIGINT）
2. 发送关闭信号给子进程
3. 等待子进程完成（最多 5 秒）
4. 关闭 HTTP 服务器
5. 关闭 Redis 连接
6. 关闭数据库连接
7. 退出进程

---

## 九、关键特性

### 9.1 安全特性
- JWT + Redis 双层认证
- 密码加密存储
- 权限控制（RBAC）
- 输入验证

### 9.2 性能优化
- Redis 缓存库存
- 原子操作避免锁
- 子进程异步处理
- 数据库连接池

### 9.3 数据一致性
- Redis 原子操作保证库存一致性
- 数据库事务保证订单创建一致性
- 异步同步机制
